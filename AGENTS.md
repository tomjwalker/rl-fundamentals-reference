# Maintainer Notes

- Treat this as a teaching repo first. Keep edits small and preserve the hand-written style of the course code.
- Do not remove or rewrite `HOMEWORK`, `ASSIGNMENT`, or `SOLUTION` markers unless the task is explicitly about student-repo generation.
- Student repos are generated from `utils/prepare_student_repo.py`. Keep root-only maintainer files, including this one, out of generated student repos.
- Prefer bug fixes, checks, and documentation improvements over broad refactors.
- Store student-facing update drafts under `communications/` as email-ready markdown files with a date-prefixed filename.
- When drafting a student update, include setup changes, student workflow changes, confirmed bug fixes, and any actions students need to take.
